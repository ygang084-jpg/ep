# 잇다 (Itda) — 취업 경험정리앱 디자인 문서

> 흩어진 경험을 모아, 나만의 자소서로.
> 프로젝트·대외활동·인턴 경험을 기록하고 STAR로 정리한 뒤, 자소서 문항에 딱 맞게 연결하는 취준생용 경험정리 앱.

- **파일**: `잇다 경험정리앱.dc.html` (Design Component, 정적 목업)
- **대상 사용자**: 대학생·취준생 (신입)
- **플랫폼**: 반응형 (모바일 우선 · 390×844 기준)
- **톤앤매너**: 신뢰감 있고 편안하며, 응원하는
- **단계**: 1단계 정적 목업 (9개 화면) → 2단계 인터랙티브 프로토타입 예정

## 4. 컴포넌트

- **폰 셸**: 390×844, `border-radius: 46px`, 부드러운 틸 그림자, 상단 스테이터스바(9:41) + 하단 홈 인디케이터
- **하단 탭바**: 홈 · 경험 · 분석 · 자소서 · MY (활성 = Primary 틸 fill + 라벨)
- **카드**: `border-radius: 16–22px`, 흰 배경, 그림자 최소화
- **칩/태그**: pill (`999px`), 역량 태그는 `# 협업` 형태 프라이머리 틴트
- **버튼**: 높이 54–56px, `border-radius: 16px` — Primary(틸 fill) / 세컨더리(다크) / 텍스트
- **STAR 카드**: 좌측 4px 컬러 바 (S 틸 · T 틸그린 · A 코랄 · R 미완성 대시)
- **진행 바 / 도넛**: 역량 커버리지 시각화 (conic-gradient 도넛, 가로 바)
- **FAB**: 우하단 56px 원형, Primary, `+` 아이콘
- **아이콘**: 인라인 SVG 스트로크 세트 (홈·리스트·차트·문서·유저·검색·체크·플러스·스파클 등), 이모지 미사용

## 5. 화면 구성 (9개)

| # | 화면 | 핵심 |
|---|------|------|
| 1 | 온보딩 | 브랜드 심볼, 응원 헤드라인, 시작 CTA (다크 틸 배경) |
| 2 | 홈 대시보드 | 인사, 정리 진행률 히어로, 역량 커버리지·진행 자소서 미니 스탯, 마감 임박 카드, 최근 경험 |
| 3 | 경험 라이브러리 | 검색, 카테고리 필터 칩, 경험 카드(상태 배지), FAB |
| 4 | 경험 기록 | 카테고리 선택, 제목·기간·한 줄 요약, 역량 태그(최대 3), 저장→STAR |
| 5 | STAR 구조화 | S/T/A/R 4단 카드, 완료 체크, R 미작성 → AI 추천 CTA |
| 6 | 역량 분석 | 커버리지 도넛(67%), 역량별 가로 바, 부족 역량 추천 카드 |
| 7 | 자소서 문항 | 기업별 카드, 문항 상태(완료/작성 중/대기), D-day |
| 8 | 경험 연결·AI 초안 | 문항 ↔ 경험 연결, STAR 기반 AI 초안, 문장 다듬기 제안 칩 |
| 9 | 마이페이지 | 프로필, 경험/STAR/지원사 스탯, 목표 기업, 설정 메뉴 |

## 6. 카피 톤

- 신뢰감 있고 편안하며 응원하는 어조 — "이번 주도 한 걸음 나아갔어요", "반가워요, 지현님"
- 부담 없이, 그러나 전문적으로: 지표·숫자는 실제 취준 맥락(CAC 22% 절감, 설문 100건 등)으로 구체화
- 이모지 미사용 (브랜드 성격상 절제)

---

## 7. 레이아웃 원칙

- 인라인 스타일 기반 (즉시 렌더), Flex/Grid + `gap` 스페이싱
- 넉넉한 여백과 낮은 채도의 민트 배경으로 편안한 밀도
- 캔버스 모드: 9개 화면을 나란히 배치, 각 화면 상단 번호 라벨로 지칭 가능
